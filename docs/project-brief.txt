1. Software you need

VS Code with Codex is your editor and AI programming assistant. It is not enough by itself because it cannot execute, compile, or serve a modern web application.

Install the following:

Required
Node.js 24 LTS

Node.js runs the development server, installs packages, builds the website, and executes server-side code. Next.js currently requires Node.js 20.9 or newer, so Node.js 24 LTS is a suitable choice. npm is installed automatically with Node.js.

After installation, confirm it in the VS Code terminal:

node --version
npm --version
Git

Use Git for version control, restoring previous versions, working with Codex safely, and deploying through GitHub.

A GitHub account

Store the project repository privately and connect it to your hosting provider.

A modern browser

Chrome or Edge is sufficient for testing and using developer tools.

Optional
Figma — wireframes and visual design.
PostgreSQL — do not install it locally initially; use a hosted database when required.
Docker Desktop — unnecessary for the first version.
Postman — useful later for testing APIs, but not required.
Image editor — useful for compressing before-and-after photographs.
Recommended stack

For this website, I would use:

Frontend and backend: Next.js
Language: TypeScript
Styling: Tailwind CSS
UI components: shadcn/ui
Forms: React Hook Form + Zod
Database: Supabase PostgreSQL
Email: Resend or SMTP
Payments: Stripe Checkout, if required
Hosting: Vercel or Railway
Source control: GitHub

Next.js can run as a complete Node.js application, a Docker container, or a limited static export. You therefore do not need a separate NestJS backend for the first version.

Supabase supplies a hosted PostgreSQL database and can later provide authentication and file storage.

Stripe Checkout can be introduced later for deposits, one-time cleaning payments, or recurring plans without creating an entire payment interface from scratch.

Create the project

Open a terminal in the folder where you keep projects:

npx create-next-app@latest cleaning-services

Choose:

TypeScript: Yes
ESLint: Yes
Tailwind CSS: Yes
src directory: Yes
App Router: Yes
Turbopack: Yes
Import alias: @/*

Then:

cd cleaning-services
npm run dev

Open:

http://localhost:3000

create-next-app is the official and simplest method for creating a new Next.js application.

2. Professional cleaning-services website

The website has three fundamental jobs:

Establish trust.
Explain exactly what is cleaned and how much it costs.
Convert the visitor into a call, quotation request, or booking.

Everything that does not support one of these objectives is secondary.

Recommended business model

The architecture should support both immediate customers and customers who are not yet ready to book.

Visitor
  ↓
Selects service
  ↓
Checks coverage, process and approximate price
  ↓
Requests quote or books inspection
  ↓
Business receives structured request
  ↓
Business confirms final price and appointment

For most cleaning companies, an instant-price calculator should initially produce an estimate, not a binding final price. Extremely dirty properties, parking problems, large windows, specialist materials, pets, furniture, and access restrictions can substantially change the required work.

Visual design
Design direction

Use a clean, premium, reassuring appearance—not a website covered in bubbles, cartoon mops, and generic stock photographs.

Recommended visual identity:

Primary colour:       Deep navy         #102A43

Typography:

Headings: Manrope or Plus Jakarta Sans
Body: Inter

Visual rules:

Large white spaces.
Rounded cards, but not excessively rounded.
Real team and project photographs.
Before-and-after comparison blocks.
Strong contrast and clearly visible buttons.
Minimal animation.
Mobile-first layout.
Consistent icon set.
Avoid carousels in the hero section.
Avoid text placed over visually complicated photographs.

Technical architecture
Recommended MVP
Browser
   ↓
Next.js application
   ├── Public website
   ├── Service pages
   ├── Quote calculator
   ├── Contact forms
   ├── API routes / server actions
   └── Basic admin protection
          ↓
Supabase PostgreSQL
          ↓
Email notification service

This is a modular monolith: one project, one deployment, one codebase.

That is superior to creating separate frontend and backend repositories at this stage. A dedicated NestJS backend would add deployment, authentication, API, validation, CORS, monitoring, and maintenance complexity without generating immediate business value.

/office-cleaning/sofia
